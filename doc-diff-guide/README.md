# 공공 PnC 가이드라인 & API 명세 정합성 검증 안내서 (원클릭 검증)

## 1. 개요

본 문서는 Gemini CLI의 강력한 워크스페이스 컨텍스트 인식 기능(아노테이션 `@` 참조)을 활용하여, `공공 V2G PnC 가이드라인` **PDF 원본 문서**와 `pnc-api-spec` 리포지토리의 전체 API 명세 간의 정합성을 **명령어 한 줄로 검증**하는 최종 절차를 안내합니다.

이 워크플로우는 별도의 사전 파일 변환이나 텍스트 복사 과정이 전혀 필요 없으며, Gemini CLI가 PDF 문서를 직접 분석하여 코드 명세와의 정합성 검증을 수행합니다.

---

## 2. 사전 준비

프로세스를 진행하기 위해 **Gemini CLI**만 설치되어 있으면 됩니다.

-   **Gemini CLI**: 터미널에서 직접 AI 검증 명령을 실행합니다.
    -   `GEMINI_API_KEY` 환경 변수가 설정되어 있어야 합니다.

---

## 3. 검증 및 검토 절차

### 단일 단계: Gemini CLI 검증 명령어 실행

가이드라인 PDF 파일이 업데이트될 때마다, 프로젝트 루트 디렉터리에서 아래 명령어 중 필요에 맞는 명령어를 터미널에 입력하여 실행합니다.

Gemini CLI가 자동으로 프로젝트의 `@api` 폴더 내 모든 파일과 루트 디렉터리의 `@공공PKI_PnC_가이드라인_V1.1_배포용.pdf` 파일을 직접 연계하여 의미론적 비교 분석을 수행합니다.

Gemini CLI는 pnc-api-spec 저장소의 루트경로에서 실행하십시오. 또한, 루트경로에 `@공공PKI_PnC_가이드라인_V1.1_배포용.pdf` 파일도 준비되어 있어야 합니다.

#### 옵션 A: 일회성 보고서 출력 후 바로 종료 (추천)
분석 결과 리포트만 화면에 출력하고 프로세스를 종료합니다.
```bash
gemini -p "$(cat doc-diff-guide/prompt.md)"
```

#### 옵션 B: 검토 결과를 출력한 후 추가 질문 및 대화 이어가기
보고서를 도출한 뒤, 특정 결과에 대해 추가적인 대화나 해결책을 연이어 묻고 싶을 때 사용합니다.
```bash
gemini -i "$(cat doc-diff-guide/prompt.md)"
```

---

## 4. 검토 결과 처리 (후속 조치)

분석이 완료되면 Gemini가 터미널에 마크다운 형식의 보고서를 출력합니다. 보고서의 **"구분"** 필드를 참고하여 한글 가이드라인(Word 등) 원본의 내용을 동기화합니다.

*   🔴 **누락**: 명세(Code)에는 존재하지만 가이드라인에 빠져 있는 신규 엔드포인트나 파라미터를 가이드라인 문서에 보강합니다.
*   🟡 **불일치**: 가이드라인에 작성된 데이터 타입, 변수명, 주체별 권한 정보 등이 코드 명세와 불일치하는 부분을 수정합니다.
*   🟢 **추가**: 가이드라인에는 적혀 있지만 코드 상에는 없는 항목입니다. 의도된 추가 사항인지 검토 후, 명세에 반영하거나 가이드라인에서 삭제 조치합니다.
