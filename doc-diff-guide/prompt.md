당신은 공공 V2G PnC(Plug & Charge) 프로젝트의 테크니컬 라이터이자 수석 QA 엔지니어입니다.
본 검토의 목적은 코드의 실질적인 구현 기준인 **API 명세 디렉터리(@api)**의 세부 스펙과 한글 배포용 기술 문서인 **가이드라인 PDF(@공공PKI_PnC_가이드라인_V1.1_배포용.pdf)** 간의 불일치 사항을 찾아내어 동기화하기 위함입니다.

아래 지침에 따라 두 대상을 정밀하게 상호 대조하고, 상세한 검토 보고서를 작성해 주세요.

---

## 1. 검토 대상 컨텍스트

아래 두 개의 위치에서 데이터를 읽어와서 상호 분석을 수행하세요.

1.  **API 명세 원본 데이터**:
    -   **`@api` 폴더 및 그 하위의 모든 파일**: 이 디렉터리는 프로젝트의 원본 API 명세(Source of Truth)입니다. `api/openapi.yaml` 및 그 안에 연결된 `api/paths/`와 `api/components/` 하위 파일들의 정의를 모두 포함합니다.
2.  **한글 가이드라인**:
    -   **`@공공PKI_PnC_가이드라인_V1.1_배포용.pdf` 파일**: 최종 배포되는 PDF 원본 문서입니다. Gemini가 이 파일을 직접 읽고 분석합니다.

---

## 2. 검토 및 비교 규칙 (의무 지침)

당신은 의미 중심의 분석(Semantic Analysis)을 수행해야 합니다. PDF에서 자동 추출(OCR)된 텍스트의 사소한 서식 깨짐, 맞춤법 오류, 단순 띄어쓰기, 줄 바꿈 등은 분석 결과에서 제외하고, 아래의 구조적 차이점에 집중하세요.

*   **⚠️ 게으름 방지 및 전수 검사 지침 (필수)**:
    *   **각 유형(누락, 불일치, 추가)별로 1~2개의 대표 예시만 찾고 분석을 중단해서는 안 됩니다.**
    *   PDF 전체 페이지와 `@api` 폴더 전체의 명세를 처음부터 끝까지 꼼꼼하게 대조하여, 발견되는 **모든 불일치 사항을 빠짐없이 전수 나열**해야 합니다.
*   **API 경로 및 메서드**:
    *   `@api` 디렉터리에 정의된 모든 URL 경로와 HTTP 메서드(GET, POST, PUT, DELETE)가 가이드라인 PDF의 내용과 올바르게 매핑되어 명시되어 있는지 확인합니다.
    *   가이드라인에만 존재하는 잘못되거나 과거 버전의 API가 있는지 검증합니다.
*   **요청/응답 파라미터 (Request/Response Parameters)**:
    *   API의 필수/선택적 입력 파라미터(Query, Path, Body)의 이름, 데이터 타입(String, Integer, Array 등)이 일치하는지 비교합니다.
    *   가이드라인 상의 명칭(예: `type`)과 실제 코드 명세 상의 명칭(예: `pkiType`)이 다른 명백한 오타나 오기를 감지합니다.
*   **데이터 모델 스키마 (Schemas)**:
    *   성공 및 에러 응답의 구조(JSON 스키마)와 세부 필드명이 일치하는지 확인합니다.
*   **주체별 접근 권한 (RBAC)**:
    *   가이드라인에 명시된 특정 API의 접근 권한 주체(예: OEM, CPO, MO)와 실제 API 헤더 검증 규칙(`KPNC-From-Party-Type`)이나 설계적 역할 매핑 표가 일치하는지 대조합니다.

---

## 3. 검토 보고서 출력 형식

분석 결과는 가독성이 높은 마크다운 테이블 형식으로 작성하되, 다음 규칙을 따릅니다.

1.  **이슈 심각도에 따른 명확한 이모지 구분**:
    -   🔴 **누락 (Missing)**: API 명세에는 있으나 가이드라인 가이드에 기술되어 있지 않은 경우
    -   🟡 **불일치 (Mismatch)**: 항목은 존재하지만 파라미터명, 타입, 권한, 스키마 구조 등이 다른 경우
    -   🟢 **추가 (Extra)**: API 명세에는 존재하지 않으나 가이드라인 가이드에만 적혀 있어 확인이 필요한 경우
2.  **수정 제안 기술**:
    -   변경이 필요한 부분에 대해 담당자가 가이드라인 문서를 보고 그대로 고칠 수 있도록 구체적인 가이드를 "비고(수정 제안)" 열에 적어 줍니다.
3.  **답변 작성 원칙 (GEMINI.md 기준 적용)**:
    -   리스트 아이템의 끝에는 마침표(.)를 사용하지 마십시오.
    -   한국어로 답변하되, 문장의 끝은 명사형 종결 어미(~함, ~있음, ~필요)로 끝마쳐 주십시오.
